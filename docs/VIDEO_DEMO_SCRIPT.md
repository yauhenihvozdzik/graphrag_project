# Сценарий видео-демонстрации — GraphRAG Platform

**Длительность**: ~10-12 минут
**Формат**: Запись экрана + голосовой комментарий

---

## Часть 1: Введение (0:00 – 1:00)

**Экран**: Слайд с названием проекта

> «Здравствуйте! Представляю вашему вниманию GraphRAG Platform — корпоративную платформу для интеллектуального анализа знаний на основе технологии Graph RAG.
>
> Платформа объединяет графовые базы данных, векторный поиск, S3-хранилище и большие языковые модели для работы с корпоративными документами. Ключевая особенность — многоуровневый контроль доступа RBAC на уровне узлов графа.»

**Действие**: Показать схему архитектуры из README.md

---

## Часть 2: Стек технологий (1:00 – 2:00)

**Экран**: терминал с `docker compose ps`

> «Платформа включает 13 Docker-сервисов:
> - **Ollama** — локальный LLM-сервер с моделью T-lite-it-1.0 (7B), дообученной T-Bank для русского языка
> - **Neo4j 5.26** — графовая база для хранения сущностей и связей
> - **Qdrant** — векторная база для семантического поиска с RBAC-фильтрацией
> - **PostgreSQL 16** — хранение пользователей, сессий и LangGraph-чекпоинтов
> - **MinIO** — S3-совместимое хранилище документов
> - **FastAPI + LangGraph** — бэкенд с оркестрацией RAG-пайплайна
> - **Prometheus + Grafana + Jaeger** — мониторинг и трассировка
> - **Mailpit + pgAdmin + Open WebUI** — вспомогательные сервисы»

**Действие**: Показать `docker compose ps` — все 13 сервисов running

---

## Часть 3: Запуск (2:00 – 3:00)

**Экран**: Терминал

> «Запуск платформы осуществляется одной командой.»

```bash
docker compose up -d
./scripts/init.sh
```

> «Скрипт init.sh автоматически запускает инфраструктуру, ожидает готовности сервисов, загружает модели Ollama (t-lite:7b-q4_K_M и bge-m3), создаёт индексы Neo4j, seed-пользователей и отделы, затем запускает backend и frontend.»

**Действие**: Показать процесс запуска и health check (`curl localhost:8000/api/v1/health`)

---

## Часть 4: Аутентификация и RBAC (3:00 – 4:00)

**Экран**: Frontend — экран входа (http://localhost:3000)

> «Система поддерживает несколько ролей пользователей с разными уровнями доступа. Войдём как аналитик.»

**Действие**:
1. Открыть http://localhost:3000
2. Ввести: `analyst@graphrag.local` / `Analyst123!`
3. Нажать "Войти"
4. Показать, что роль отображается в sidebar (Аналитик, отдел legal)
5. Показать доступные вкладки: Чат, Загрузка, Документы
6. Выйти и войти как admin (`admin@graphrag.local` / `Admin123!`)
7. Показать расширенный sidebar: Пользователи, Отделы, Тесты, Инфраструктура

> «Viewer видит только чат, аналитик — чат, загрузку и документы, администратор — полный набор включая управление пользователями, отделами и тесты.»

---

## Часть 5: Загрузка документов (4:00 – 5:30)

**Экран**: Вкладка "Загрузка" (под admin)

> «Загрузим юридический документ в систему. Система автоматически:
> 1. Разобьёт текст на чанки
> 2. Извлечёт сущности — организации, законы, персоны
> 3. Построит связи в графе знаний
> 4. Создаст векторные представления для поиска
> 5. Сохранит текст в MinIO (S3)»

**Действие**:
1. Перейти на вкладку "📄 Загрузка"
2. Ввести заголовок: "Гражданский кодекс РФ — Статья 1"
3. Вставить текст статьи
4. Выбрать уровень доступа: "2 — Confidential"
5. Выбрать отдел: "Юридический"
6. Нажать "Загрузить текст"
7. Показать успешное сообщение с document_id

> «Также поддерживается загрузка файлов: PDF, DOCX, TXT, MD, ZIP. Файлы сохраняются в MinIO, что позволяет скачать оригинал позже.»

---

## Часть 6: Чат с GraphRAG (5:30 – 7:30)

**Экран**: Вкладка "Чат"

> «Теперь зададим вопросы по загруженным документам. Система использует гибридный поиск: векторный + графовый.»

**Действие и вопросы**:

1. **Простой вопрос**:
   > Введём: `Какие принципы гражданского законодательства?`

   Показать ответ с Markdown-форматированием и источниками (📎).

2. **Вопрос о связях**:
   > `Как связаны право собственности и свобода договора?`

   Показать, что система использует граф для поиска связей.

3. **Демонстрация RBAC в действии**:
   > Открыть новое приватное окно, войти как `viewer@graphrag.local`.
   > Задать тот же вопрос — показать, что ответ ограничен уровнем доступа (только public-документы).

> «Обратите внимание: viewer видит только открытые документы, тогда как аналитик имел доступ к конфиденциальным. Фильтрация работает на уровне узлов графа и векторного поиска.»

## Часть 7: Документы и граф знаний (7:30 – 8:30)

**Экран**: Вкладка "Документы" и "Граф знаний"

> «Во вкладке "Документы" мы видим таблицу загруженных документов с пагинацией, фильтрами и возможностью скачать содержимое.»

**Действие**:
1. Перейти на вкладку "📚 Документы"
2. Показать таблицу с фильтрами по отделу и уровню доступа
3. Нажать "Скачать" на документе — показать, что контент отдаётся как файл

6. Открыть Neo4j Browser (через вкладку "Инфраструктура")
7. Выполнить Cypher-запрос:
   ```cypher
   MATCH (e:Entity)-[r]->(e2:Entity) RETURN e, r, e2 LIMIT 50
   ```
8. Показать визуальный граф с узлами и связями

## Технические заметки для записи

- **Разрешение экрана**: 1920x1080
- **Браузер**: Chrome (тёмная тема)
- **Терминал**: Windows Terminal с тёмной темой
- **Шрифт**: увеличенный до 16-18px для читаемости
- Перед записью убедитесь, что все сервисы запущены: `docker compose ps` (должно быть 13 сервисов)
- Предварительно загрузите модели Ollama: `t-lite:7b-q4_K_M` и `bge-m3`
- Создайте пользователей через `python scripts/seed_users.py` и отделы через `python scripts/seed_departments.py`
- Опционально загрузите демо-данные через `python scripts/load_datasets.py`
- Для демонстрации безопасности заранее загрузите несколько документов с разными уровнями доступа
- Подготовьте аккаунты в разных браузерах/окнах: admin, analyst, viewer
- Убедитесь, что Mailpit работает (http://localhost:8025) для показа email-уведомлений
