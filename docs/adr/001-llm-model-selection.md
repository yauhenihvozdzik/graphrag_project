# ADR-001: Выбор LLM модели для GraphRAG платформы

## Статус
Принято

## Контекст

Платформа GraphRAG для анализа корпоративных знаний требует LLM модель, способную:
- Качественно работать с русскоязычными юридическими текстами (датасеты RusLawOD, RFSD)
- Полностью помещаться в 8 GB VRAM (NVIDIA RTX 4060)
- Работать on-premise без облачных API (air-gapped среда)
- Обеспечивать достаточную скорость генерации для интерактивного использования

**Аппаратные ограничения:**
- GPU: RTX 4060 — 8 GB VRAM, Ada Lovelace архитектура
- CPU: Ryzen 3900X — 12 ядер / 24 потока
- RAM: 64 GB DDR4
- ОС: Windows 11 Pro

**Бюджет VRAM (8 GB):**
- Веса модели: ≤ 5.5 GB (для Q5_K_M) или ≤ 4.5 GB (для Q4_K_M)
- KV-cache: ~1.0–2.0 GB (контекст 4096–8192 токенов)
- Runtime overhead (Ollama/llama.cpp): ~0.5 GB
- Резерв: ~0.5 GB (ОС, другие процессы)

## Рассмотренные варианты

### 1. Qwen 2.5-7B-Instruct (GGUF Q5_K_M)
- **VRAM**: ~5.4 GB веса + ~1.5 GB KV-cache = ~7.4 GB (контекст 4096)
- **Русский язык**: Обучен на 29+ языках, включая русский. Хорошие результаты на ruMMLU, ruGSM8K
- **Скорость**: ~40–50 tok/s на RTX 4060 при полном GPU offload
- **Контекст**: до 32K токенов (практически до 8K при 8 GB VRAM)
- **Лицензия**: Apache 2.0

### 2. T-lite-it-1.0 (7B, на базе Qwen 2.5)
- **VRAM**: аналогично Qwen 2.5-7B (~5.4 GB в Q5_K_M)
- **Русский язык**: Специально дообучена T-Bank для русского через Continuous Pretraining. Превосходит оригинальный Qwen 2.5-7B на бенчмарках MERA, ruMMLU-PRO, ruGSM8K, Alpaca Eval Ru
- **Скорость**: аналогична Qwen 2.5-7B
- **Лицензия**: Apache 2.0
- **Нюанс**: модель доступна на HuggingFace, GGUF-квантования от сообщества

### 3. Qwen 2.5-14B-Instruct (GGUF Q4_K_M)
- **VRAM**: ~8.7 GB только веса — **не помещается** полностью в 8 GB
- **Частичная выгрузка**: ~28/40 слоёв на GPU, остальное в RAM
- **Скорость**: ~12–18 tok/s (значительная деградация из-за PCIe bottleneck)
- **Русский язык**: лучше 7B по качеству, но разница не критична для RAG-задач

### 4. DeepSeek-R1-Distill-Qwen-7B (GGUF Q4_K_M)
- **VRAM**: ~4.0 GB в Q4_K_M
- **Русский язык**: нет специализированных бенчмарков для русского, обучена преимущественно на английском/китайском
- **Сильные стороны**: reasoning, математика, код (AIME 55.5%, MATH-500 92.8%)
- **Слабые стороны**: не оптимальна для юридических текстов на русском

### 5. Saiga (Llama-3-8B fine-tune)
- **VRAM**: ~5.0 GB в Q4_K_M
- **Русский язык**: специально дообучена для русского, хорошие результаты в ru_llm_arena
- **Критический недостаток**: проект IlyaGusev/rulm **архивирован** (март 2025), нет активной поддержки
- **Лицензия**: зависит от базовой модели (Llama 3 Community License)

### 6. Qwen 2.5-7B-Instruct (GGUF Q4_K_M)
- **VRAM**: ~4.5 GB веса — больший запас для KV-cache
- **Русский язык**: чуть хуже Q5_K_M, но приемлемо для RAG (основной контекст в промпте)
- **Скорость**: ~45–55 tok/s
- **Запас VRAM**: ~3.0 GB на KV-cache → поддержка контекста до ~8192 токенов

## Решение

**Основная модель: T-lite-it-1.0 (7B) в квантовании GGUF Q4_K_M**

**Запасная модель: Qwen 2.5-7B-Instruct в квантовании GGUF Q4_K_M**

### Конфигурация развёртывания:

| Параметр | Значение |
|---|---|
| Модель | `t-tech/T-lite-it-1.0` (GGUF Q4_K_M) |
| VRAM (веса) | ~4.5 GB |
| VRAM (KV-cache, ctx 4096) | ~1.5 GB |
| VRAM (total) | ~6.5 GB |
| Контекст | 4096–8192 токенов |
| Скорость генерации | ~40–50 tok/s |
| GPU offload | Полный (все слои на GPU) |

## Обоснование

### Почему T-lite-it-1.0:

1. **Русский язык — главный приоритет**: T-lite специально дообучена на русскоязычных данных через Continuous Pretraining поверх Qwen 2.5. На бенчмарках MERA, ruMMLU-PRO, ruGSM8K она **превосходит** оригинальный Qwen 2.5-7B-Instruct
2. **Юридический домен**: для работы с RusLawOD и RFSD критично качество понимания русскоязычных юридических конструкций — T-lite лучше подходит
3. **Совместимость**: основана на Qwen 2.5, полная совместимость с GGUF-квантованием и Ollama
4. **Лицензия Apache 2.0**: подходит для корпоративного on-premise использования

### Почему Q4_K_M, а не Q5_K_M:

1. **Запас VRAM**: Q4_K_M (~4.5 GB) оставляет ~3.0 GB для KV-cache, что позволяет работать с контекстом 8192 токенов — критично для RAG-промптов с извлечёнными фрагментами документов
2. **Качество**: в RAG-сценарии основная информация приходит из контекста (retrieval), а не из параметрических знаний модели. Разница между Q4_K_M и Q5_K_M минимальна для задачи extraction/summarization
3. **Стабильность**: меньше риск OOM при пиковых нагрузках или длинных промптах

### Почему не 14B:

- Qwen 2.5-14B Q4_K_M (~8.7 GB) не помещается в 8 GB VRAM
- Частичная выгрузка слоёв в RAM даёт ~12–18 tok/s — **неприемлемо** для интерактивного использования
- PCIe 4.0 x8 (RTX 4060) создаёт bottleneck при передаче данных GPU↔RAM

### Почему не DeepSeek-R1-Distill:

- Оптимизирована для reasoning/математики, а не для юридических текстов
- Нет бенчмарков для русского языка
- Для RAG важнее instruction following и summarization, чем deep reasoning

### Почему не Saiga:

- Проект **архивирован**, нет гарантии исправления багов и обновлений
- Зависимость от Llama 3 Community License (ограничения для коммерческого использования)
- T-lite покрывает тот же use case с активной поддержкой

## Последствия

### Положительные:
- ✅ Полный GPU offload — максимальная скорость генерации (~40–50 tok/s)
- ✅ Специализация на русском языке — лучшее качество для юридического домена
- ✅ Запас VRAM для KV-cache — поддержка длинных RAG-промптов (до 8K токенов)
- ✅ Apache 2.0 — без лицензионных ограничений
- ✅ Совместимость с Ollama — простое развёртывание на Windows

### Отрицательные:
- ⚠️ 7B модель имеет ограничения в сложных reasoning-задачах по сравнению с 14B+
- ⚠️ Q4_K_M квантование даёт ~1–2% деградации качества по сравнению с FP16
- ⚠️ T-lite — модель от одной компании (T-Bank), возможен риск прекращения поддержки
- ⚠️ GGUF-квантования T-lite создаются сообществом, а не официально

### Митигация рисков:
- Qwen 2.5-7B-Instruct Q4_K_M как fallback — официальные GGUF от Qwen Team
- Архитектура системы позволяет заменить модель без изменения кода (Ollama абстрагирует модель за API)
- При обновлении GPU (≥12 GB VRAM) — переход на Qwen 2.5-14B Q4_K_M для лучшего качества
