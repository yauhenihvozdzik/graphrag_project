# Руководство пользователя — GraphRAG Platform

---

## 1. Вход в систему

1. Откройте http://localhost:3000 в браузере
2. На экране входа введите ваш **email** и **пароль**
3. Нажмите **"Войти"**

### Демо-аккаунты

| Email | Пароль | Роль | Отдел | Доступ |
|-------|--------|------|-------|--------|
| admin@graphrag.local | Admin123! | Администратор | management | Полный доступ ко всем документам и управление пользователями |
| analyst@graphrag.local | Analyst123! | Аналитик | legal | Загрузка + чтение документов своего уровня |
| viewer@graphrag.local | Viewer123! | Читатель | research | Только чтение открытых документов |
| legal@graphrag.local | Legal123! | Аналитик | legal | Загрузка + чтение документов своего уровня |

### Регистрация
Нажмите **"Регистрация"**, заполните имя, email и пароль (минимум 8 символов, заглавная буква, цифра).

После регистрации вы получите email об ожидании активации. Администратор должен активировать ваш аккаунт через панель управления пользователями.

## 2. Чат с GraphRAG

Вкладка **"💬 Чат"** — основной инструмент работы.

### Как задавать вопросы

Введите вопрос в поле внизу экрана и нажмите **Enter** или кнопку **"Отправить"**.

#### Примеры вопросов:
- `Какие основные принципы гражданского законодательства?`
- `Что регулирует закон о персональных данных?`
- `Какие права защищает Трудовой кодекс?`
- `Найди связи между Гражданским кодексом и правом собственности`

### Понимание ответов

Каждый ответ содержит:
- **Текст ответа** — сгенерирован на основе найденных документов (Markdown-форматирование)
- **📎 Источники** — список документов, использованных для ответа (с возможностью скачивания)

> **Важно**: Ответы генерируются на основе загруженных документов. Если информации нет в базе, система сообщит об этом.

### Контроль доступа в чате

Система автоматически фильтрует контекст по вашему уровню доступа:
- **Viewer** видит только документы уровня 0 (public/открытый)
- **Analyst** видит документы уровней 0–1 (public + internal) своего отдела
- **Admin** видит все документы

### Очистка истории
Чат-история сохраняется в PostgreSQL. Для очистки используйте кнопку очистки в интерфейсе чата — запрос `DELETE /api/v1/chat/history`.

## 3. Загрузка документов

Вкладка **"📄 Загрузка"** (доступна для ролей `analyst` и `admin`).

### Загрузка текста

1. Введите **заголовок** документа
2. Вставьте **текст** в поле "Содержимое"
3. Выберите **уровень доступа**:
   - **0 — Public** — открытый, доступен всем
   - **1 — Internal** — внутренний, для авторизованных
   - **2 — Confidential** — конфиденциальный, ограниченный доступ
   - **3 — Secret** — секретный, только для администраторов
4. Выберите **отдел** (Все, Юридический, Исследования, Управление, Комплаенс, HR, Финансы, IT)
5. Нажмите **"Загрузить текст"**

Система автоматически проверяет дубликаты по хешу содержимого.

### Загрузка файлов

1. Нажмите **"📎 Выберите файл"**
2. Выберите файл формата PDF, DOCX, TXT, MD или ZIP
3. Нажмите **"Загрузить файл"**

Файлы сохраняются в MinIO (S3-хранилище). Система проверяет дубликаты по имени и размеру файла.

### Загрузка по URL

Доступна через API: `POST /api/v1/ingest/url` — передайте URL и метаданные документа в JSON.

### Что происходит при загрузке

1. Документ разбивается на чанки (фрагменты по 512 символов с перекрытием 64)
2. Извлекаются сущности (организации, законы, персоны) — rule-based NER
3. Строятся связи в графе знаний (Neo4j: Document → Chunk → Entity)
4. Создаются векторные представления для поиска (Qdrant, 1024-dim bge-m3)
5. Текст сохраняется в S3 (MinIO) для последующего скачивания

⏱ Обработка документа занимает от 10 секунд до нескольких минут. Статус можно отслеживать через `GET /api/v1/ingest/status/{doc_id}`.

## 4. Граф знаний и документы

Вкладка **"📚 Документы"** — управление загруженными документами.

### Список документов

- Таблица со всеми документами: название, отдел, уровень доступа, количество чанков, дата
- Пагинация, сортировка
- Фильтры по отделу и уровню доступа
- Кнопка **"Скачать"** для получения содержимого документа:
  - Уровень 1: оригинальный файл из MinIO (если сохранялся)
  - Уровень 2: извлечённый текст из S3 / Neo4j full_text
  - Уровень 3: сборка из чанков

### Neo4j Browser

Встроенный Neo4j Browser позволяет:
- Исследовать граф визуально
- Выполнять Cypher-запросы
- Просматривать свойства узлов и связей

Доступ: http://localhost:7474 (neo4j / neo4j_password), либо через вкладку "Инфраструктура" во frontend.

### Базовые Cypher-запросы

```cypher
// Все сущности
MATCH (e:Entity) RETURN e LIMIT 50

// Связи между сущностями
MATCH (e1:Entity)-[r]->(e2:Entity) RETURN e1, r, e2 LIMIT 100

// Поиск по имени
MATCH (e:Entity) WHERE e.name CONTAINS 'кодекс' RETURN e

// Документы и их сущности
MATCH (d:Document)-[:CONTAINS]->(c:Chunk)-[:MENTIONS]->(e:Entity)
RETURN d, c, e LIMIT 50
```

### Статистика графа

Через API `GET /api/v1/graph/stats` или кнопку **"Обновить статистику"** во вкладке "Граф знаний":
- Количество узлов
- Количество связей
- Количество документов
- Количество векторных точек в Qdrant

### Удаление документов

Документы можно удалить:
- Через кнопку удаления в таблице документов → вызывает `DELETE /api/v1/graph/document/{doc_id}`
- Удаление затрагивает: Neo4j (узлы и связи), Qdrant (векторы), MinIO (файлы), PostgreSQL (file_metadata)
- **Полная очистка**: кнопка "Очистить всё" → `DELETE /api/v1/graph/clear` (только для admin)

## 5. Безопасность

### Что защищает система

- **ПДн (персональные данные)**: ИНН, СНИЛС, паспорта, телефоны, email, банковские реквизиты автоматически маскируются
- **Prompt injection**: Попытки манипулировать LLM блокируются (на русском и английском языках)
- **XSS/SQL injection**: Входные данные очищаются (модуль sanitization)

### Если вы видите предупреждение безопасности

Это означает, что ваш запрос содержит паттерны, похожие на попытку атаки. Переформулируйте запрос более конкретно.

## 6. Администрирование (только admin)

Вкладка **"👥 Пользователи"** доступна только администратору.

### Управление пользователями

- **Таблица пользователей**: пагинация, сортировка, фильтры по роли/отделу/уровню доступа/email
- **Изменение**: роль, отдел, уровень доступа, активация/деактивация
- **Активация**: новый пользователь получает email после регистрации; admin активирует его — пользователь получает email об активации
- **Удаление**: admin может удалить любого пользователя (кроме себя)

### Имперсонация

Администратор может войти под любым пользователем (кнопка "👤 Войти как"):
- В frontend сохраняется оригинальный admin-токен
- При выходе из имперсонации admin возвращается в свой аккаунт
- API: `POST /api/v1/auth/users/{id}/impersonate`

### Управление отделами

Вкладка **"🏢 Отделы"** (admin): создание, редактирование, удаление отделов.

### Запуск тестов

Вкладка **"🧪 Тесты"** (admin): запуск pytest через SSE с просмотром результатов в реальном времени.

## 8. Часто задаваемые вопросы

**Q: Почему система не находит информацию?**
A: Убедитесь, что документы загружены. Проверьте уровень доступа — возможно, документ ограничен для вашей роли или отдела.

**Q: Почему ответ генерируется медленно?**
A: Скорость зависит от модели LLM и доступных ресурсов (CPU/GPU). Первый запрос обычно медленнее. Основная модель T-lite-it-1.0 (7B) работает на GPU через Ollama.

**Q: Можно ли загружать документы на английском?**
A: Да, но система оптимизирована для русскоязычных юридических документов. Embedding-модель bge-m3 поддерживает многоязычный поиск.

**Q: Как удалить загруженный документ?**
A: Через кнопку удаления в таблице документов (вкладка "📚 Документы"). Удаление затрагивает все слои: Neo4j, Qdrant, MinIO, PostgreSQL. Полная очистка графа: кнопка "Очистить всё".

**Q: Сколько документов можно загрузить?**
A: Ограничений нет, но производительность зависит от объёма данных и ресурсов сервера. Рекомендуется мониторить использование RAM через Grafana.

**Q: Какие форматы файлов поддерживаются?**
A: PDF, DOCX, TXT, MD, ZIP. Список определяется в `backend/app/core/prompts.py` (`ALLOWED_FILE_EXTENSIONS`).

**Q: Хранятся ли оригиналы загруженных файлов?**
A: Да, оригиналы сохраняются в MinIO (S3-совместимое хранилище). При скачивании документа оригинальный файл имеет приоритет над извлечённым текстом.
